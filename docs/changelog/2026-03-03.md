## 1. ステータス設計の整理：DB 保存値と表示値の分離

### 概要

従来は `scheduled / active / ended` などの「表示用ステータス」をそのまま DB に保存していたが、DB 保存用（3値）と表示用（5値）を型レベルで分離。

### 詳細

| ファイル | 変更内容 |
|---------|---------|
| `types/database.ts` | `CupDbStatus = 'draft' \| 'published' \| 'finalized'` を新設。`CupStatus` は表示用（5値）として存続 |
| `lib/repositories/cup.ts` | `withComputedStatus` を適用。`published` を期間から `scheduled / active / ended` に自動変換して返す |
| `lib/repositories/cup.ts` | `findAll()` に `toDbStatusFilter()` を導入。表示値 → DB値への変換 + メモリ内再フィルタの二段方式 |
| `components/admin/CupForm.tsx` | ステータス選択肢を `draft / published / finalized` の3択に整理 |
| `app/admin/cups/[id]/page.tsx` | ステータス管理の操作ボタンを DB 値の3択に整理。期間から算出されるステータスは「（期間から自動算出）」と注釈表示 |
| `app/api/cups/[id]/route.ts` | `VALID_DB_STATUSES` の許可リストで不正ステータス値を拒否 |

## 3. カップ更新 API のセキュリティ・バリデーション強化

### 確定済みカップの変更ロック

- `PATCH /api/cups/[id]` で既存カップを事前取得し、`status === 'finalized'` なら 403 を返す
- 管理フォームでも `isFinalized` フラグで `<fieldset disabled>` にして編集不可、警告バナー表示

### 日付バリデーション

- `end_at < start_at` の場合、`POST`・`PATCH` の両方でサーバー側 400 エラー
- クライアント側（`CupForm.tsx`）でも事前チェック + リアルタイム表示
- 終了日の `min` 属性に開始日を自動設定

### エラーログの改善

- 500 エラー時に `console.error` でサーバーログ出力
- エラーメッセージをレスポンスに含めるように改善

## 4. 参加状況 API の新設と統合

### 新規 API

`app/api/me/participations/route.ts` — `GET /api/me/participations`

- `wallet_session` Cookie から `profile_id` を検証
- `cup_participants` から参加中の `cup_id` 一覧を返却
- 未ログイン時は空配列（エラーにしない設計）

### フロントエンド統合

| ファイル | 変更内容 |
|---------|---------|
| `components/cup/CupLobby.tsx` | カップ一覧と参加情報を `Promise.all` で並行取得。`registeredCupIds: Set<string>` を管理 |
| `components/cup/CupCard.tsx` | `isRegistered` プロップ追加。参加済み → 「参加済み」ボタン、未参加 → 「参加登録」ボタン。`scheduled` 状態ではランキングボタン非表示。カード全体を `<Link>` でクリッカブルに |
| `app/(user)/cup/[id]/page.tsx` | 詳細ページでも参加状況を取得。`canRegister` の条件を `scheduled` のみに修正（大会開始後は参加不可 = 仕様準拠） |

### タブラベル変更

`CupLobby.tsx` — 「予定」→「参加受付中」、「完了」→「開催終了」
