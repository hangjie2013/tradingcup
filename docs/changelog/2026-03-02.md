## 1. バナー機能の新規実装（フルスタック）

### 概要

ロビー画面のヒーロー部分を動的なキャンペーンバナーに置き換え。管理画面からバナーの作成・有効/無効切替・削除が可能。

### DBスキーマ

`supabase/migrations/004_banners.sql` で `banners` テーブルを作成。

| カラム | 型 | 説明 |
|--------|-----|------|
| `id` | UUID | PK |
| `image_key` | TEXT NOT NULL | Storage の object_key（URL直保存ではない） |
| `link_url` | TEXT | バナークリック時の遷移先 |
| `sort_order` | INT | 表示順 |
| `is_active` | BOOLEAN | 有効/無効 |
| `created_at` | TIMESTAMPTZ | 作成日時 |

- RLS: `SELECT` のみパブリック公開
- `banners` Storage バケットも同時作成

### Repository 層

| ファイル | メソッド |
|---------|---------|
| `lib/repositories/banner.ts` | `findActive` / `findAll` / `create` / `update` / `delete` |

### Storage 層 (`lib/storage/index.ts`)

- `uploadBanner` / `getBannerUrl` メソッドを追加
- バケット定数を `CUP_COVERS_BUCKET` / `BANNERS_BUCKET` に分離

### API ルート

| ファイル | メソッド | 説明 |
|---------|---------|------|
| `app/api/banners/route.ts` | `GET` | アクティブバナー1件を公開取得（認証不要） |
| `app/api/admin/banners/route.ts` | `GET` / `POST` | 管理者向け一覧取得・新規作成 |
| `app/api/admin/banners/[id]/route.ts` | `PATCH` / `DELETE` | 有効/無効切替・削除（PATCHはホワイトリスト制） |

### 管理画面

| ファイル | 内容 |
|---------|------|
| `app/admin/banners/page.tsx` | バナー一覧表示、有効/無効トグル、削除 |
| `app/admin/banners/new/page.tsx` | 画像アップロード（MIME・サイズ検証）+ リンクURL設定 |
| `components/admin/AdminHeader.tsx` | 管理画面共通ヘッダー（カップ戦・バナー・アカウントへのナビ + ログアウト） |

### フロントエンド

## 2. 認証状態管理のリファクタリング（AuthContext 導入）

### 概要

各コンポーネントが個別にセッションチェック API を呼んでいた構造を、グローバルな `AuthContext` に一元化。

### 新規ファイル

`lib/auth/AuthContext.tsx`

```typescript
interface AuthState {
  isAuthenticated: boolean
  isChecking: boolean
  walletAddress: string | null
  profileId: string | null
  setAuthenticated: (walletAddress: string, profileId?: string) => void
  clearAuth: () => void
  recheckSession: () => Promise<void>
}
```

- `useAccount`（wagmi）の接続状態を監視し、変化時に自動セッション確認
- ウォレット切断時は自動リセット

### 影響ファイル

| ファイル | 変更内容 |
|---------|---------|
| `app/providers.tsx` | `AuthProvider` を `RainbowKitProvider` 直下に追加 |
| `components/wallet/ConnectButton.tsx` | ローカルステート → `useAuth()` に移行。チェック中はローディングスピナー表示 |
| `app/(user)/profile/page.tsx` | `useAccount` → `useAuth()` に移行 |
